Detailed Description

EVMC – Ethereum Client-VM Connector API.

High level design rules

  1. Pass function arguments and results by value. This rule comes from modern C++ and tries to avoid costly alias analysis needed for optimization. As the result we have a lots of complex structs and unions. And variable sized arrays of bytes cannot be passed by copy.
  2. The EVM operates on integers so it prefers values to be host-endian. On the other hand, LLVM can generate good code for byte swaping. The interface also tries to match host application "natural" endianess. I would like to know what endianess you use and where.

Terms

  1. EVM – an Ethereum Virtual Machine instance/implementation.
  2. Host – an entity controlling the EVM. The Host requests code execution and responses to EVM queries by callback functions.

This callback function is used by an EVM to request a copy of the code of the given account to the memory buffer provided by the EVM. ◆ evmc_status_code

enum evmc_status_code

The execution status code.

Enumerator
EVMC_SUCCESS 

Execution finished with success.

EVMC_FAILURE 

Generic execution failure.

EVMC_UNDEFINED_INSTRUCTION 

Unknown instruction encountered by the VM.

EVMC_REVERT 

Execution terminated with REVERT opcode.

EVMC_STATIC_MODE_VIOLATION 

Tried to execute an operation which is restricted in static mode.

EVMC_INVALID_INSTRUCTION 

The dedicated INVALID instruction was hit.

EVMC_INVALID_MEMORY_ACCESS 

Tried to read outside memory bounds.

An example is RETURNDATACOPY reading past the available buffer.

EVMC_PRECOMPILE_FAILURE 

Exceptions produced by precompiles/system contracts.

An example: elliptic curve functions handed invalid EC points

EVMC_CALL_DEPTH_EXCEDED 

Call depth exceded (if there is a call depth limit)

EVMC_CONTRACT_VALIDATION_FAILURE 

Contract validation has failed (e.g.

due to EVM 1.5 jump validity, Casper's purity checker or ewasm contract rules).

EVMC_REJECTED 

The EVM rejected the execution of the given code or message.

This error SHOULD be used to signal that the EVM is not able to or willing to execute the given code type or message. If an EVM returns the EVMC_REJECTED status code, the Client MAY try to execute it in other EVM implementation. For example, the Client tries running a code in the EVM 1.5. If the code is not supported there, the execution falls back to the EVM 1.0.

EVMC_INTERNAL_ERROR 

EVM implementation internal error.

Todo:
We should rethink reporting internal errors. One of the options it to allow using any negative value to represent internal errors.
